01 What it is

One console for every repo and every agent.

Lab is a workbench, not a chat box. You open it and see every project you work on as a card. Behind each card is a real working tree, a git remote, and (when you want one) a live coding agent in its own terminal session.

It runs the agents you already use: Claude Code and Codex, each in its own PTY with full scrollback, drag-and-drop file context, and saved prompts a keystroke away. Spin up a side session to explore while the main one keeps working.

The leverage is in the fan-out. Write a macro once and run it across ten checkouts at a time, with a concurrency cap so your machine stays usable. When the work lands, Lab stages, commits, and pushes, then watches the GitHub Actions run that follows.

This is Directed Coding: you direct a crew of agents the way a film director directs a set (supervisory and intentional) instead of hand-typing every line into a single conversation.

02 How you direct

Brief the fleet. Watch it work. Ship what's good.

  1. 01 · BRIEF

    Hand out the work

    Fire a saved prompt or a macro at one project or fan it across many. Each repo gets its own agent session, rooted in its own checkout, running in parallel under a cap you set.

  2. 02 · DIRECT

    Stay in the loop

    Every session is live and interactive: jump in, nudge, course-correct, drop in a file for context. Lab pings you the moment an agent is waiting on you, so you direct the ones that need it.

  3. 03 · SHIP

    Review, commit, push

    Read the diff, stage and commit with one click, push, and watch the CI run that follows, across all the repos you touched, without leaving the console.
